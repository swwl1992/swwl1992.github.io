Why I used Jekyll instead of Octopress

13 June 2013

I saw the beautiful blog made by my friend and I was amazed. However I faced a lot of problems when I installed both Jekyll and Octopress. Below is some examples:

Building native extensions.  This could take a while...
ERROR:  Error installing jekyll:
ERROR: Failed to build gem native extension.

/usr/bin/ruby1.9.1 extconf.rb
/usr/lib/ruby/1.9.1/rubygems/custom_require.rb:36:in `require': cannot load such file -- mkmf (LoadError)
from /usr/lib/ruby/1.9.1/rubygems/custom_require.rb:36:in `require'
from extconf.rb:1:in `<main>'

Solution: sudo apt-get install ruby1.9.1-dev

Bundler::GemfileNotFound

Solution: find the Gemfile by command: locate Gemfile cd into the directory

rake aborted!
You have already activated rake 0.9.6, but your Gemfile requires rake 0.9.2.2.

Solution: run command $ bundle exec rake install Apply to all the incidents when rake is aborted.

Then the final boss came and I was unable to configure the rake such that it can point to my github local repo. Sad…

Now I switch to Jekyll Bootstrap. Hopefully it is a better solution. Hopefully it was not because I was not technical enough.
